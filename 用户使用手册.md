## 🎯 系统概述

### 1.1 系统简介

TOPSIS作战评估系统是一个基于多准则决策分析的作战效能评估工具，使用TOPSIS（Technique for Order Preference by Similarity to Ideal Solution）算法对四种典型作战场景进行量化评估。

### 1.2 核心功能

- **四种作战场景评估**：海上封锁、登陆场清扫、水下监视、水雷清排
- **智能化参数配置**：根据场景自动调整参数输入界面
- **科学化评估算法**：基于TOPSIS算法的多准则决策分析
- **可视化结果展示**：雷达图、柱状图、仪表盘等多种图表
- **智能化建议生成**：基于评估结果提供改进建议

### 1.3 适用对象

- 军事指挥人员
- 作战研究人员
- 军事院校师生
- 国防科技工作者

## 🔧 功能详解

### 4.1 作战场景模块

#### 4.1.1 海上封锁作战
**场景描述**：评估海上封锁作战方案的效能，包括对敌方海上交通线的封锁能力。

**主要参数**：
- 舰艇数量：参与封锁的作战舰艇总数
- 潜艇数量：用于水下封锁的潜艇数量
- 飞机数量：提供空中支援和侦察的飞机数量
- 导弹射程：反舰导弹的最大有效射程
- 通信系统等级：指挥通信系统的技术水平

**评估重点**：火力打击能力、隐蔽能力、持续作战能力。

#### 4.1.2 登陆场清扫作战
**场景描述**：评估登陆场清扫作战方案的效能，包括清除登陆场障碍和敌方防御的能力。

**主要参数**：
- 登陆艇数量：用于登陆作战的登陆艇总数
- 工兵分队：负责清除障碍的工兵分队数量
- 炮兵支援：提供火力支援的炮兵单位数量
- 空中优势：确保制空权的战斗机数量
- 扫雷能力：清除水雷的能力等级

**评估重点**：登陆能力、障碍清除能力、火力支援能力。

#### 4.1.3 水下监视作战
**场景描述**：评估水下监视作战方案的效能，包括对水下目标的探测和监视能力。

**主要参数**：
- 海上巡逻：海上巡逻舰艇数量
- 反潜直升机：用于反潜作战的直升机数量
- 声呐系统：水下探测系统的技术水平
- 监视范围：水下监视的有效范围
- 数据链：数据通信系统的技术水平

**评估重点**：探测能力、监视能力、反应速度。

#### 4.1.4 水雷清排作战
**场景描述**：评估水雷清排作战方案的效能，包括清除水雷障碍的安全性和效率。

**主要参数**：
- 扫雷舰艇：专业扫雷舰艇数量
- 潜水员：水下作业的潜水员数量
- 扫雷设备：扫雷设备的技术水平
- 安全措施：安全保障措施的完善程度
- 作业效率：清排作业的效率等级

**评估重点**：清排能力、安全保障、作业效率。

### 4.2 参数配置模块

#### 4.2.1 参数输入界面
- **左侧显示**：参数名称和描述说明
- **右侧显示**：数值输入框和单位
- **实时验证**：输入时自动检查参数范围

#### 4.2.2 参数类型说明
```
类型：整数型
示例：舰艇数量
范围：1-50
默认值：10
单位：艘

类型：浮点型
示例：导弹射程
范围：10-1000
默认值：200
单位：公里

类型：等级型
示例：通信系统等级
范围：1-10
默认值：5
单位：级
```

#### 4.2.3 参数验证规则
- **范围检查**：确保参数值在有效范围内
- **类型检查**：确保输入正确的数据类型
- **逻辑检查**：确保参数间的逻辑关系合理

### 4.3 评估算法模块

#### 4.3.1 TOPSIS算法流程
1. **参数量化**：将输入参数转换为标准化指标值
2. **权重分配**：根据指标重要性分配权重
3. **标准化处理**：消除量纲影响
4. **理想解计算**：确定正理想解和负理想解
5. **距离计算**：计算各方案到理想解的距离
6. **相对贴近度**：计算综合评价指数
7. **结果排序**：根据贴近度进行排名

#### 4.3.2 五大核心指标
- **综合作战能力**：反映作战力量的整体水平（权重30%）
- **作战效能**：作战任务的执行效率（权重25%）
- **作战适应性**：适应不同作战环境的能力（权重20%）
- **作战保障**：后勤保障能力（权重15%）
- **协同能力**：多军兵种协同作战能力（权重10%）

### 4.4 结果展示模块

#### 4.4.1 数值结果
```
综合得分：0.742
评估等级：B（良好）
相对排名：2/5
与优秀方案差距：-0.108
```

#### 4.4.2 可视化图表
1. **雷达图**：显示五大核心指标的得分情况
2. **柱状图**：对比当前方案与参考方案的得分
3. **仪表盘**：直观显示综合得分水平

#### 4.4.3 改进建议
基于评估结果自动生成针对性改进建议，例如：
- "建议增加舰艇数量以提高火力打击能力"
- "通信系统等级有待提升，建议升级通信设备"
- "空中支援能力相对薄弱，建议增加飞机数量"

## 📈 结果解读

### 6.1 评分体系

#### 6.1.1 得分范围
- **得分区间**：0.0 - 1.0
- **优秀**：0.8 - 1.0
- **良好**：0.6 - 0.8
- **一般**：0.4 - 0.6
- **较差**：0.0 - 0.4

#### 6.1.2 等级划分
```
A级（优秀）：0.8 ≤ 得分 ≤ 1.0
B级（良好）：0.6 ≤ 得分 < 0.8
C级（一般）：0.4 ≤ 得分 < 0.6
D级（较差）：0.0 ≤ 得分 < 0.4
```

### 6.2 排名系统

#### 6.2.1 相对排名含义
系统将当前方案与4个参考方案进行比较：
- **优秀方案**：理论最优配置
- **良好方案**：实际可行的高水平配置
- **一般方案**：基本满足需求的配置
- **较差方案**：存在明显不足的配置

#### 6.2.2 排名解读
```
排名：1/5 - 超越所有参考方案
排名：2/5 - 超越良好、一般和较差方案
排名：3/5 - 超越一般和较差方案
排名：4/5 - 仅超越较差方案
排名：5/5 - 需要显著改进
```

### 6.3 指标分析

#### 6.3.1 雷达图解读
雷达图显示了五大核心指标的得分情况：
- **外圈**：表示满分（1.0）
- **内圈**：表示零分（0.0）
- **多边形**：当前方案各指标的实际得分
- **面积**：多边形面积越大，整体表现越好

#### 6.3.2 指标改进优先级
根据雷达图分析，优先改进得分较低的指标：
1. **得分 < 0.4**：急需改进
2. **得分 0.4-0.6**：建议改进
3. **得分 0.6-0.8**：适度改进
4. **得分 > 0.8**：保持优势

### 6.4 改进建议解读

#### 6.4.1 建议类型
1. **参数调整建议**：直接建议调整特定参数值
2. **能力提升建议**：建议提升某方面的能力
3. **资源配置建议**：建议优化资源分配
4. **技术升级建议**：建议升级技术设备

#### 6.4.2 建议优先级
```
高优先级：影响综合作战能力的关键因素
中优先级：提升整体效能的重要因素
低优先级：完善细节的优化因素
```

## ❓ 常见问题

### 7.1 安装和启动问题

#### Q: 安装时提示"权限不足"怎么办？
**A**:
- Windows: 右键点击命令提示符，选择"以管理员身份运行"
- macOS/Linux: 使用 `sudo pip install` 命令
- 或者使用用户级安装：`pip install --user`

#### Q: 启动时提示"端口已被占用"怎么办？
**A**:
1. 查找占用端口的进程：`netstat -ano | findstr :8501`
2. 结束该进程或更改端口：`streamlit run app.py --server.port 8502`

#### Q: 浏览器无法访问系统怎么办？
**A**:
1. 确认Streamlit服务正在运行
2. 检查防火墙设置
3. 尝试使用不同的浏览器
4. 确认URL地址正确

### 7.2 参数配置问题

#### Q: 参数输入框显示为红色怎么办？
**A**:
- 输入值超出了允许范围
- 输入了错误的数据类型
- 系统正在实时验证输入值

#### Q: 如何设置合理的参数值？
**A**:
- 参考"典型配置示例"
- 考虑实际装备情况
- 均衡配置各项参数
- 避免极端值设置

#### Q: 参数设置后是否可以保存？
**A**:
- 当前版本参数仅在本次会话中有效
- 关闭浏览器后参数将重置为默认值
- 建议记录重要的参数配置

### 7.3 评估结果问题

#### Q: 评估得分总是很低怎么办？
**A**:
- 检查参数设置是否合理
- 参考改进建议调整参数
- 考虑资源配置的均衡性
- 逐步优化参数设置

#### Q: 相对排名总是第二名正常吗？
**A**:
- 这是正常现象，系统参考方案设置合理
- 第二名表示方案质量良好
- 可以通过优化参数争取第一名

#### Q: 不同场景的评估结果如何比较？
**A**:
- 建议在同一场景内比较不同方案
- 不同场景的评估标准不同，不宜直接比较
- 可以关注各场景的相对排名

### 7.4 系统性能问题

#### Q: 评估计算时间很长怎么办？
**A**:
- 正常计算时间为3-5秒
- 如果时间过长，可能是系统资源不足
- 建议关闭其他占用内存的程序
- 重启浏览器或系统

#### Q: 图表显示不完整怎么办？
**A**:

## 📚 附录

### 9.1 技术术语

#### 9.1.1 TOPSIS算法
**定义**: TOPSIS（Technique for Order Preference by Similarity to Ideal Solution）是一种多准则决策分析方法，通过计算方案与理想解的相对贴近度来进行排序。

**特点**:
- 同时考虑正理想解和负理想解
- 计算简单，结果直观
- 适用于多属性决策问题

#### 9.1.2 多准则决策分析
**定义**: 在多个 conflicting 准则下，从有限方案中选择最优方案的方法。

**应用**:
- 军事决策
- 工程设计
- 经济评估
- 环境保护

#### 9.1.3 雷达图
**定义**: 一种显示多变量数据的图表，可以直观地显示各指标的相对表现。

**特点**:
- 可以显示多个维度
- 便于比较不同方案
- 直观显示优势与不足

### 9.2 数学公式

#### 9.2.1 标准化公式
```
r_ij = x_ij / √(∑x_ij²)
其中：
- x_ij 为原始参数值
- r_ij 为标准化后的值
- i 为方案编号，j 为指标编号
```

#### 9.2.2 加权标准化矩阵
```
v_ij = w_j × r_ij
其中：
- w_j 为第j个指标的权重
- v_ij 为加权标准化值
```

#### 9.2.3 欧几里得距离
```
D+_i = √[∑(v_ij - v+_j)²]
D-_i = √[∑(v_ij - v-_j)²]
其中：
- D+_i 为到正理想解的距离
- D-_i 为到负理想解的距离
- v+_j 为正理想解的第j个值
- v-_j 为负理想解的第j个值
```

#### 9.2.4 相对贴近度
```
C_i = D-_i / (D+_i + D-_i)
其中：
- C_i 为第i个方案的相对贴近度
- C_i 值越大，方案越优
```
